# Лабораторная работа: Реализация методов массивов с использованием колбэков

## Цель работы

Изучение принципов реализации базовых методов работы с массивами с использованием колбэков: `forEach`, `map`, `filter`, `find`, `some`, `every`, `reduce`, а также закрепление навыков написания функций с корректным возвратом результатов и обработкой граничных случаев.

## Ход выполнения работы

В ходе лабораторной работы были реализованы следующие функции без использования встроенных методов массивов:

| Функция | Описание | Возвращает |
|---|---|---|
| `printArray` | Вывод массива в формате `Element i: value x` | `undefined` |
| `printArray1` | Вывод массива в формате `i: x` | `undefined` |
| `forEach` | Выполняет колбэк для каждого элемента | `undefined` |
| `map` | Преобразует каждый элемент через колбэк | Новый массив |
| `filter` | Оставляет элементы, прошедшие проверку | Новый массив |
| `find` | Находит первый подходящий элемент | Элемент или `undefined` |
| `some` | Проверяет наличие хотя бы одного совпадения | `boolean` |
| `every` | Проверяет, все ли элементы прошли проверку | `boolean` |
| `reduce` | Сворачивает массив в одно значение | Аккумулятор |

Все функции реализованы с использованием цикла `for`, свойства `length`, обращения по индексу и метода `push`. Код задокументирован по стандарту JSDoc.

## Контрольные вопросы

### 1. В чём преимущества использования колбэков при работе с массивами?

Колбэки позволяют отделить **логику обхода** массива от **логики обработки** каждого элемента. Это даёт несколько преимуществ:

- **Переиспользование кода** — одна функция `forEach` или `map` может применяться с любой логикой, переданной через колбэк, не нужно писать новый цикл каждый раз.
- **Читаемость** — код становится декларативным: вместо `for (let i = 0; ...)` пишем `map(arr, el => el * 2)`, что сразу понятно по смыслу.
- **Разделение ответственности** — функция-обёртка отвечает за обход, колбэк — за бизнес-логику.
- **Гибкость** — одну и ту же функцию `filter` можно использовать и для фильтрации чисел, и строк, и объектов, просто передав нужный колбэк.

### 2. Какие проблемы могут возникать при использовании колбэков и как их избежать?

| Проблема | Описание | Решение |
|---|---|---|
| Передача не-функции | Если в `callback` передать число или строку, получим `TypeError` | Добавить проверку `typeof callback === 'function'` |
| Контекст `this` | Внутри колбэка `this` может указывать не туда, куда ожидается | Использовать стрелочные функции `() =>` |
| Мутация исходного массива | Колбэк может случайно изменить оригинальный массив | Не изменять `array[i]` внутри колбэка, работать с копиями |
| Бесконечная рекурсия | Колбэк вызывает ту же функцию | Проектировать колбэки без побочных эффектов |
| Пустой массив | `reduce` без `initialValue` на пустом массиве вернёт `undefined` | Обрабатывать граничный случай явно |

### 3. Как реализовать функции `map`, `filter`, `find`, `some`, `every` и `reduce` без встроенных методов?

Все функции строятся по единому шаблону на базе цикла `for`:

```
function имяФункции(array, callback) {
    for (let i = 0; i < array.length; i++) {
        // вызываем callback(array[i], i, array)
        // в зависимости от результата — добавляем в массив, возвращаем значение и т.д.
    }
    // возвращаем результат
}
```

Ключевые отличия реализаций:

- **`map`** — всегда добавляет результат колбэка в новый массив через `push`, длина результата равна длине исходного.
- **`filter`** — добавляет элемент только если колбэк вернул `true`, длина результата меньше или равна исходной.
- **`find`** — при первом `true` сразу делает `return array[i]`, не продолжая обход.
- **`some`** — при первом `true` возвращает `true`, в конце цикла возвращает `false`.
- **`every`** — при первом `false` (через `!callback(...)`) возвращает `false`, в конце цикла возвращает `true`.
- **`reduce`** — хранит аккумулятор, обновляя его на каждой итерации: `accumulator = callback(accumulator, array[i], i, array)`.

## Выводы

В ходе работы была изучена концепция колбэков как механизма передачи логики обработки в универсальные функции обхода массива. Реализация функций `map`, `filter`, `find`, `some`, `every` и `reduce` через цикл `for` наглядно показала, что все встроенные методы массивов основаны на одном принципе — итерации с вызовом переданной функции для каждого элемента. Использование колбэков делает код гибким и переиспользуемым, а понимание их внутреннего устройства помогает грамотно применять встроенные методы на практике.
